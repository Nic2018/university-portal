# User Guide

## 9.1 Login to System

1. User access to login page.
2. User enters Student ID/Username and password in the login form.
3. User press "Sign in to Portal" button.
4. If the user wants to access as a guest, they can click "Continue as Guest" button to access read-only features.
5. If the user doesn't have an account, they can click "Create an account" link to register.
6. User is logged in and redirected to their respective dashboard.

## 9.2 Logout from System

1. User clicks the "Log Out" button at the bottom of the sidebar.
2. User is redirected to the login page.

## 9.3 Search Space Availability

1. User clicks the "Find" button on the dashboard or the Pin icon in the sidebar.
2. User views the "Find Your Space" page with a grid of available venues.
3. User views venue details including capacity, equipment, and location.
4. User can enter keywords or select filters (Location/Size) to refine search results.
5. User clicks "Book Now" on a specific venue card.
6. User is redirected to the booking form with the selected room pre-filled.

## 9.4 Request Booking

1. User clicks the "+" button in the sidebar or the "Book" card on the dashboard.
2. User views the "Book a Venue" form.
3. User selects booking purpose from dropdown (Study Session or Event).
4. User selects desired venue from the venue dropdown.
5. User enters event name and optional description.
6. User lists any add-on equipment needed (optional).
7. User selects start time and end time using the date/time picker.
8. User clicks "Confirm Booking" button.
9. User is redirected to dashboard and booking status shows as "Pending".

## 9.5 Manage Own Booking

1. User clicks "My Bookings" card on the dashboard or the calendar icon in the sidebar.
2. User views a list of all bookings with event name, venue, date, time, and status.
3. User can filter bookings by status (All, Pending, Approved, Rejected).
4. User clicks on a booking to view full details.
5. User clicks "Cancel" button to cancel a booking.
6. User confirms cancellation and the booking is removed from the list.

## 9.6 View Campus Map

1. User clicks the "Map" card on the dashboard or the map icon in the sidebar.
2. User views an interactive campus map showing building locations.
3. User can zoom in/out and pan around the map.
4. User clicks on building markers to see venues within.
5. User views venue information by clicking on venue pins.

## 9.7 View Schedule

1. User clicks the "Calendar" or "Schedule" card on the dashboard.
2. User views an interactive calendar showing the current month.
3. User can navigate between months using previous/next arrows.
4. User clicks on any date to view events scheduled for that day.
5. User can click "Book This Date" to initiate a booking with the date pre-selected.

## 9.8 View Event Details

1. User navigates to the schedule/calendar view.
2. User clicks on a date that has events.
3. User selects an event from the list.
4. User views comprehensive event information including event name, organizer, venue, date, time, description, capacity, equipment, and status.
5. User clicks "Close" or "Back" to return to the calendar view.

## 9.9 Manage Personal Profile

1. User clicks their profile picture or username in the top-right corner.
2. User views current profile information.
3. User clicks "Edit Profile" button.
4. User modifies desired fields (name, email, phone number, profile picture).
5. To change password, user clicks "Change Password" tab.
6. User enters current password, new password, and confirms new password.
7. User clicks "Save Changes" button.
8. User receives confirmation message.

## 9.10 Campus Bot

1. User clicks the chat bubble icon in the bottom-right corner.
2. User views the chat window with a welcome message.
3. User types a question in the text input field.
4. User presses Enter or clicks the Send button.
5. User receives bot response with relevant information, links, or quick action buttons.
6. User can continue the conversation with follow-up questions.
7. User clicks the X button to close the chat window.

## 9.11 Admin Login

1. Administrator access to admin login page.
2. Administrator enters admin username and password.
3. Administrator clicks "Admin Login" button.
4. Administrator is authenticated and redirected to the admin dashboard with access to all administrative functions.

## 9.12 View and Manage Venues

1. Administrator clicks "Venues" or "Manage Venues" from the admin dashboard.
2. Administrator views a list of all venues with room code, name, location, capacity, equipment, and status.
3. Administrator can search for specific venues or apply filters by location, capacity, equipment, or status.
4. Administrator clicks column headers to sort the venue list.
5. Administrator clicks on any venue to view full details and booking history.

## 9.13 Edit Venue Details

1. Administrator navigates to "Manage Venues" section.
2. Administrator finds the desired venue and clicks the "Edit" button.
3. Administrator views the venue editing form.
4. Administrator modifies venue information (name, room code, location, capacity, description).
5. Administrator updates the equipment list by adding or removing items.
6. Administrator sets availability status (Active/Inactive).
7. Administrator uploads venue photos (optional).
8. Administrator clicks "Save Changes" button.
9. Administrator receives confirmation message.

## 9.14 Manage All Bookings

1. Administrator clicks "Bookings" or "Manage Bookings" from the admin dashboard.
2. Administrator views a comprehensive list of all bookings with event name, organizer, venue, date, time, and status.
3. Administrator can filter bookings by status (Pending, Approved, Rejected), date range, venue, or organizer.
4. Administrator clicks on a pending booking to review full details.
5. To approve: Administrator clicks "Approve" button, adds optional notes, and confirms approval.
6. To reject: Administrator clicks "Reject" button, enters rejection reason, and confirms rejection.
7. To modify: Administrator clicks "Edit" button, changes details (time, venue, equipment), and saves changes.
8. To cancel: Administrator clicks "Cancel" button, enters cancellation reason, and confirms cancellation.

## 9.15 Manage Groups and Permissions

1. Administrator clicks "Groups & Permissions" from the admin dashboard.
2. Administrator views all user groups/roles (Student, Event Organizer, Guest, Administrator, Staff).
3. Administrator clicks on a group to view its permissions.
4. To create new group: Administrator clicks "Add New Group", enters group name and description, and clicks "Create".
5. To assign permissions: Administrator selects a group, checks/unchecks permission boxes for venue management, booking management, user management, and system settings, then clicks "Save Permissions".
6. To assign users to group: Administrator clicks "Add Members", searches for users, selects them, and clicks "Add to Group".
7. To remove users: Administrator clicks "Remove" next to a user in the group members list.

## 9.16 Manage Users

1. Administrator clicks "Users" or "Manage Users" from the admin dashboard.
2. Administrator views a list of all registered users with username, name, email, role, status, and registration date.
3. Administrator can search for users by name, student ID, or email, and apply filters by role, status, or registration date.
4. To create new user: Administrator clicks "Add New User", fills in registration form (username, name, email, phone, department, password, role), and clicks "Create User".
5. To edit user: Administrator finds the user, clicks "Edit", modifies information (name, email, phone, department, role), and clicks "Save Changes".
6. To reset password: Administrator clicks "Reset Password", chooses auto-generate or custom password option, and clicks "Reset".
7. To deactivate user: Administrator clicks "Deactivate" or toggles status switch.
8. To delete user: Administrator clicks "Delete", confirms by typing username, and clicks "Permanently Delete".
